# Guía de incidente 01 — El dashboard perdió la conexión

**Para:** Staff y directores de brigada  
**Cuándo usar esta guía:** El dashboard muestra un mensaje de error o dejó de actualizarse solo

---

## Qué está pasando

El dashboard del área se conecta a internet para recibir actualizaciones en tiempo real. Si la conexión se interrumpe, el dashboard se congela y muestra un aviso de error. Los datos de la brigada **no se pierden** — todo está guardado en el sistema.

---

## ¿Cómo sé si pasó esto?

Verás este mensaje en la pantalla:

> **"Conexión perdida. Por favor recarga la página para continuar."**

O bien: la cola dejó de actualizarse aunque sí se están atendiendo pacientes.

---

## Qué hacer — paso a paso

### Paso 1 — Recarga la página

En la tablet o teléfono, recarga el navegador:
- **Android / Chrome:** Toca los tres puntos (⋮) → "Recargar"
- **iPhone / Safari:** Toca el ícono de recarga (⟳) en la barra de direcciones
- **Computadora:** Presiona `F5` o `Ctrl + R`

Espera 5 segundos. El aviso de error debería desaparecer y la cola debería mostrar el estado actual.

✅ **Si el dashboard vuelve a funcionar, listo.** Continúa operando normalmente.

---

### Paso 2 — Si recargar no funciona

Verifica que el dispositivo tenga internet:
- Abre otra aplicación o página web en el mismo dispositivo.
- Si tampoco carga, el problema es la red WiFi o datos del dispositivo — no el sistema.
- Conéctate a otra red WiFi o activa los datos móviles e intenta recargar de nuevo.

---

### Paso 3 — Si el internet funciona pero el dashboard sigue sin conectarse

El sistema puede estar presentando una falla temporal. Mientras se resuelve:

- **Continúa operando la cola normalmente** — cada vez que presiones "Siguiente", "Mover al final" o "Eliminar", la acción sí se guarda aunque el dashboard no se actualice solo.
- **Recarga la página cada 1 o 2 minutos** para ver el estado actual.
- **Avisa al director de brigada** para que esté al tanto.

---

### Paso 4 — Si el problema persiste más de 10 minutos

Contacta al administrador de la plataforma. Comparte:
- La hora aproximada en que empezó el problema.
- El nombre de la brigada y el área afectada.
- Qué dispositivo y red estás usando.

---

## Lo que NO debes hacer

- ❌ No cierres la brigada por este motivo — los datos están seguros.
- ❌ No dejes de atender pacientes — las acciones de cola siguen funcionando.
- ❌ No registres pacientes dos veces pensando que se perdieron — todo está guardado.

---

## Recuerda

Una desconexión del dashboard **no borra ningún dato**. En cuanto recargues la página, verás el estado real y actualizado de la cola.
